# 统一 AI Papers 实施计划

日期：2026-07-12  
依据：2026-07-12-unified-ai-papers-design.md

## 目标

在不修改 ob-ai-papers 与 ur-ai-papers 的前提下，交付一个本地可运行、可验证的新项目：

- OB、UR 正式发布数据进入统一索引；
- 相同论文跨频道去重，但频道评分与标签分别保留；
- 心理健康具备配置、免费候选抓取和受控 LLM 评分骨架；
- Next.js 网站按已确认界面展示统一论文流；
- 测试、构建与浏览器冒烟验证通过。

## 工作包

### A. 统一发布层

1. 定义 canonical key 与 URL-safe stable id。
2. 从旧仓库指定 git revision 读取正式发布 JSON。
3. 归一化 OB、UR 字段。
4. 合并跨频道重复论文，保留 channel_profiles 与 source_refs。
5. 导出列表、逐论文详情、元数据、更新状态和审计报告。
6. 测试 DOI、arXiv、标题指纹、冲突与幂等。

### B. 心理健康频道

1. 落地来源、召回、排除、评分和标签配置。
2. 实现 OpenAlex 小批量免费候选抓取。
3. 实现默认 dry-run、显式限额的 LLM 双评分。
4. 输出与统一发布层一致的频道数据。
5. 测试双信号门槛、误召回、无密钥与空输入。

### C. 前端

1. 建立 Next.js App Router 项目。
2. 实现统一数据加载与错误状态。
3. 实现频道、搜索、动态筛选、排序和清空。
4. 实现最近发表、收藏、已读、详情与关于页面。
5. 实现桌面侧栏和手机筛选抽屉。
6. 落地三频道光谱线与论文卡领域轨道。
7. 通过生产构建和可访问性基础检查。

### D. 集成与验收

1. 使用 OB、UR 正式 revision 构建统一索引。
2. 免费抓取心理健康候选，输出候选量与运行报告。
3. 在不触发全量付费回填的前提下，决定是否运行极小 LLM 验证批次。
4. 重新生成统一数据并构建前端。
5. 运行 Python 测试、Next.js 构建和桌面/手机浏览器冒烟测试。
6. 记录已知限制与下一阶段选择。

## 完成判定

- 旧项目工作树未被本项目改写；
- 新项目 git 工作树仅包含本轮预期文件；
- 统一论文数与各频道输入、去重报告相互一致；
- 页面使用真实 OB、UR 数据，心理健康状态不伪装为已完成全量回填；
- 所有自动化验证通过，或明确记录真实阻塞与影响。
