## 已完成

- 确认“打开即统一论文流”的产品结构。
- 确认冷灰研究索引视觉方向与三频道光谱标识。
- 完成并提交设计规格。
- 确定首版采用“OB / UR 正式发布数据 + MH 独立流水线 + 统一发布层”。
- 明确稳定 ID、跨频道去重、最近发表与 RSS 的语义边界。
- 完成 OB / UR 正式数据适配、稳定 ID 和跨频道去重。
- 完成心理健康 29 个计划来源配置；22 个成功抓取，7 个无法精确解析而安全跳过。
- 免费抓取得到 46 篇候选，12 篇通过双信号预检。
- MiniMax 小批量评分 12 篇，9 篇达到双阈值并进入正式论文流，0 失败。
- 2026-07-13 从两个上游 `origin/main` 刷新后：1,940 条频道记录，排除 12 条 2023 年前记录，合并为 1,895 篇论文。
- 当前 32 篇属于跨频道论文；真实标识符冲突进入审计报告，不静默覆盖。
- 完成统一首页、组合筛选、最近发表、收藏、详情、关于、RSS 和移动筛选抽屉。
- 参照 OB AI Papers 与 UR AI Papers，完成全站紧凑密度调整：统一收紧标题行高、区块上下距、卡片内边距、筛选控件、详情页、关于页、页脚与移动端覆盖样式。
- 将四个领域入口归入右侧论文结果列，与搜索、标题和卡片共用同一栅格；左侧筛选栏不再被领域导航横跨。
- 完成桌面双栏视觉分区：272px 浅灰筛选底板、60px 统一顶部模块和贯穿论文流的中央分隔线；移动端自动恢复单列。
- 将桌面搜索框归入左侧筛选栏；移除论文卡摘要与标签之间的分割线，并进一步压缩卡片内边距、标签间距和卡片间距。
- 将排序控件移入论文流标题栏右侧，删除与左侧统计重复的论文数量，并移除原排序工具行以缩短频道栏到论文列表的距离。
- 21 项 Python 测试、TypeScript 类型检查、Next.js 生产构建与生产浏览器冒烟测试通过。
- 浏览器复验无 console 错误、无 HTTP 错误。
- 完成频道注册表驱动：频道名称、顺序、颜色、必需状态和数据目录由 `config/channels.yaml` 统一管理；前端与发布层不再只接受三个固定频道。
- 新增第四频道自动发布测试，证明新增频道无需修改发布器、校验器或 React 频道分支。
- 完成一键更新器：OB / UR 远端同步、逐频道失败回退、心理健康可选受控更新、统一构建、RSS 和发布校验串成一个命令。
- 完成跨文件发布校验，覆盖稳定 ID、计数、详情文件、频道注册、coverage、data_version 与 RSS。
- 新增 GitHub CI、北京时间每日更新工作流、Vercel 配置与 Makefile 验证入口。
